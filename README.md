## 1. Quick Start

> All commands use the `ml` conda environment. Prefix every command with `PYTHONIOENCODING=utf-8` on Windows.

```bash
ML=/c/Users/Prasanna/miniconda3/envs/ml/python.exe

# Step 0 — find the optimal number of clusters
$ML find_optimal_k.py
# Open results/optimal_k_analysis.html, pick your k

# Step 1 — build DuckDB from parquet
PYTHONIOENCODING=utf-8 $ML -m clustering_pipeline.cli build-db

# Step 2 — train (skip stratify when total stained tiles < ~500k)
PYTHONIOENCODING=utf-8 $ML -m clustering_pipeline.cli train \
    --table-name embeddings --k 16

# Step 3 — predict cluster labels for all stained tiles
PYTHONIOENCODING=utf-8 $ML -m clustering_pipeline.cli predict \
    --k 16 --model-path models/kmeans_model_k16__above_all.joblib

# Step 4 — merge with extended metadata
PYTHONIOENCODING=utf-8 $ML -m clustering_pipeline.cli postprocess

# Step 5 — HTML analysis reports
PYTHONIOENCODING=utf-8 $ML -m clustering_pipeline.cli analyze \
    --k 16 --pred-db-path db/predictions_extended.db

# Step 6 — UMAP scatter (stratified, cluster centroids plotted)
$ML projection_plot.py --method umap

# Step 7 — image grids per cluster from local tiles
$ML cluster_image_grid.py
```

## 4. Algorithm — MiniBatchKMeans

### Objective

Standard K-Means minimises **within-cluster sum of squared Euclidean distances**:

$$\min \sum_{i=0}^{k-1} \sum_{x \in C_i} \|x - \mu_i\|^2$$

where $x \in \mathbb{R}^{384}$ is a tile embedding, $\mu_i$ is the centroid of cluster $i$, and $C_i$ is the set of tiles assigned to cluster $i$.

**Assignment rule** — each tile is assigned to its nearest centroid:

$$\text{cluster}(x) = \arg\min_{i} \|x - \mu_i\|^2$$

### MiniBatch Variant

Instead of computing over the full dataset per iteration, `MiniBatchKMeans` draws random mini-batches and updates centroids with an exponential moving average — making it suitable for millions of tiles while using constant memory.

### Low-Tissue Relabeling (optional, when `tissue_percentage` is available)

After prediction, postprocessing can reassign special cluster IDs:

| `tissue_percentage` | `cluster` |
|---|---|
| `< 5` | `-2` |
| `5 – 20` | `-1` |
| `≥ 20` | original cluster |

> **Note**: `global_color_histograms.parquet` does **not** contain `tissue_percentage`, so relabeling is skipped automatically; original cluster labels are kept.

## 5. Finding the Optimal k

`find_optimal_k.py` tests a range of k values on a stratified sample and produces:

- **Elbow curve** — inertia (WCSS) vs k; the bend indicates diminishing returns
- **Silhouette scores** — measures how well-separated clusters are (higher = better)

```bash
# Default: tests k ∈ [8,16,24,32,48,64,80,96,112,128,160,200] on 50k stained tiles
python find_optimal_k.py

# Custom
python find_optimal_k.py --k-values 8 16 32 64 128 --sample 30000 --modality stained
```

Outputs:
- `results/optimal_k_analysis.html` — interactive Plotly chart with elbow + silhouette
- `results/optimal_k_results.csv` — raw numbers

**Interpretation**: elbow method picks the k where inertia stops decreasing sharply. Silhouette scores are often low in high-dimensional embedding spaces — the elbow is usually the more reliable guide.

## 6. DuckDB Utilities


- Creates an `embeddings` table from the flat parquet

```python
create_duckdb_from_parquet_simple(

    generate_unique_id=False,
    index_unique_id=False,
)
```

Used to build `extended_data.db` from `global_color_histograms.parquet`.

### `duckdb_iter_batches`

```python
duckdb_iter_batches(table, batch_size, columns=None, min_tissue_pct=0, modality="stained")
```

Streams batches from `embeddings.db`.

- `modality` — filter by modality (default `"stained"`); parameterised (previously hardcoded)
- `min_tissue_pct` — if > 0, joins `ext_db.extended_data` and filters `tissue_percentage`

## 7. Stratified Sampling

```python
create_stratified_dataset(
    parquet_paths,          # list of parquet paths or "duckdb:<table>" strings
    stratify_field="tissue_type",
    modality="stained",
    per_class=25_000,       # max tiles per tissue type
    max_samples=100_000,    # global cap
    write_to_duckdb=False,
    embed_db_path=None,
    extended_db_path=None,
)
```

Two-pass algorithm:
1. **Pass 1** — count rows per class per source
2. **Pass 2** — sample proportionally up to quota, tracking `seen_ids` to avoid duplicates

> **When to skip**: if stained tiles < ~500k, train directly on the `embeddings` table with `--table-name embeddings`.

## 8. Training

### From DuckDB (recommended)

```python
fit_kmeans_model_duckdb(
    embed_db_path, extended_db_path,
    table_name="embeddings",
    k=16,
    batch_size=100_000,
    min_tissue_pct=None,   # None = no tissue filter
    modality="stained",
    suffix=None,
    out_dir="./models",
)
```

Streams random batches directly from DuckDB and calls `MiniBatchKMeans.partial_fit`. Saves `models/kmeans_model_k{k}[_suffix].joblib`.

**Model naming**: the CLI auto-generates a suffix from `--sample-nb` and `--min-tissue-pct`. For a run with no tissue filter and no sample limit, the model will be named `kmeans_model_k{k}__above_all.joblib`. Always check `models/` and pass `--model-path` explicitly to the `predict` step.

### From DataFrame

```python
fit_kmeans_model(df, k, batch_size=100_000, suffix=None, out_dir="./models")
```

For experiments on smaller, pre-loaded DataFrames.

## 9. Prediction

```python
predict_kmeans_model_duckdb(
    model_path, embed_db_path, pred_db_path,
    table_name="embeddings",
    batch_size=100_000,
    modality="stained",
    out_parquet_path=None,
)
```

- Uses an **anti-join** (`LEFT JOIN … WHERE p.unique_id IS NULL`) to fetch only unpredicted rows — making prediction **idempotent and resumable**
- Writes `(unique_id, id, slide_key, tile_key, short_box_name, box_id, cluster)` to `pred_db.predictions`
- Optionally exports to Parquet at `out_parquet_path`

## 10. Postprocessing

```python
create_predictions_extended_db(
    pred_db_path, extended_db_path, output_db_path,
    pred_table="predictions",
    ext_table="extended_data",
)
```

- Joins predictions with extended metadata on `tile_key`
- Applies low-tissue relabeling **only if** `tissue_percentage` exists in `ext_table` (graceful fallback otherwise)
- Deduplicates the join using `WHERE modality = 'stained'` to avoid double-rows when extended_data contains both stained and unstained rows per tile
- Prints cluster count deltas before/after relabeling
- Creates indexes on `unique_id` and `tile_key`

## 12. Visualization — Image Grids

### `cluster_image_grid.py` (local tiles)

```bash
python cluster_image_grid.py [--rows 5] [--cols 5] [--tile-size 224]
```

Reads tiles from the local filesystem using the path formula:
```
{tiles_root}/{tissue_type}/{block_id}/{slice_id}/{scan_date}/{short_box_name}/{filename}-stained.tiff
```

> Uses `short_box_name` for the folder (e.g. `16400-0`), **not** `box_id` (e.g. `64551`).

Outputs:
- `results/cluster_grids/cluster_{id}.png` — labelled 5×5 grid per cluster
- `results/cluster_grids/overview.html` — self-contained HTML page with all grids

### `visualize_cluster_samples_duckdb` (S3 tiles)

```python
visualize_cluster_samples_duckdb(
    embed_db_path, pred_db_path, extended_db_path,
    table_name="predictions",

    rows=10, cols=10,

```

Downloads tiles from S3/R2 via `S3Downloader` (multi-threaded), builds OpenCV grids, saves PNGs. Requires R2 credentials in `r2.conf`.

## 13. Visualization — UMAP / t-SNE

`projection_plot.py` produces an interactive scatter coloured by cluster.

```bash
# UMAP — recommended (faster, better global structure)
python projection_plot.py --method umap --per-cluster 600

# t-SNE — alternative
python projection_plot.py --method tsne --per-cluster 600 --perplexity 50
```

**Key improvements over a naive t-SNE**:

| Issue | Fix |
|---|---|
| Random sampling under-represents small clusters | **Stratified sampling**: equal tiles per cluster (default 600) |
| t-SNE distorts global structure | **UMAP** preserves both local and global topology |
| No visual reference for cluster positions | **Centroid stars** plotted on top of the scatter |

Pipeline: PCA 384D → 130D (95% variance retained, auto-selected) → UMAP/t-SNE 130D → 2D.

Output: `results/{method}_k16_stratified.html`
